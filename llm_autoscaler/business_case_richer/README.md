# business_case_richer/

Real-cluster business-case figures from `results_richer/` (the N=3-replicated,
two-trace sweep: `cpu_bursty` + `wiki_diurnal`). Generated via
`plot_business_case_real.py` and `plot_percentiles_real.py` against
`results_richer/{cpu_bursty,wiki_diurnal}`.

**Differs from `../business_case_real/`**, which holds the earlier
`results_k8s_v2` figures (single rep, no `gpt-oss-120b`, no wiki_diurnal
breakout). Do not conflate the two — file naming here always ends in
`_bursty` or `_wiki` to make the source trace explicit; the old directory's
files end in plain `_cpu`/`_io`.

Snapshot as of 2026-07-17 09:20 UTC: cpu_bursty is 8/9 controllers at full
N=3 (Qwen3-80B partial, being retired); wiki_diurnal is partial (see
`summary_wiki_diurnal.csv` / `percentiles_wiki_diurnal.csv` for exact reps
completed per config). Note: GPT-OSS-120B's `cot` variant on cpu_bursty
shows a reproducible scale-oscillation bug (P99 ~3.8s, 63% success) across
all 3 reps — visible as the extreme outlier in the ladder/tail-ratio charts.

## `ablation_llm_overhead.{csv,png}` — latency now vs. estimated if local

Generated by `../ablation_llm_overhead.py`. Two numbers per model, plus the
reduction:

| model | latency now (real API) | estimated if local | reduction |
|---|---|---|---|
| Mistral-Small | 1.8s | 0.5s | 1.3s (72%) |
| Llama-8B | 2.2s | 0.6s | 1.6s (74%) |
| GPT-OSS-120B | 14.3s | 0.6s | 13.7s (96%) |
| Llama-70B | 52.2s | 26.0s | 26.3s (50%) |
| Qwen3-80B | 143.3s | 80.4s | 62.9s (44%) |

`latency_now_s` is the mean of the measured `llm_latency_ms` column
(cpu_bursty, all reps/variants). `latency_if_local_s` is **an estimate, not
a measurement** — we have not run any of these models on local GPUs. It's
the *median* of that same model's own latency distribution: half of all
real API calls already land at or below it, i.e. it's what a "nothing went
wrong" call looks like (no rate-limit wait, no 504 retry, no
provider-fallback rotation). Local hosting removes exactly those things,
so the median is a reasonable floor for a dedicated local deployment.

It is **not** a floor on compute time itself — Llama-70B's and Qwen3-80B's
estimates are still tens of seconds, because a large model takes real GPU
time to generate regardless of where it's hosted. The big wins (GPT-OSS-120B
at 96%, Llama-8B/Mistral in the 70s%) are models whose current latency is
mostly network/provider overhead on top of a fast model; the smaller wins
(Llama-70B 50%, Qwen3-80B 44%) are models where the "if local" number is
still large because the model itself is slow to run.
